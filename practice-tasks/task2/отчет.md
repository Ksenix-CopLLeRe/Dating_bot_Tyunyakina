# СРАВНЕНИЕ RABBITMQ И REDIS

## 1. Введение

Цель работы. Сравнить Redis и RabbitMQ как брокеры сообщений.

Сравнение проводится по метрикам:

-   пропускная способность\
-   задержка доставки\
-   потери сообщений

## 2. Описание стенда

Стенд состоит из трёх частей:

-   producer. Отправляет сообщения\
-   broker. Redis или RabbitMQ\
-   consumer. Получает сообщения и считает метрики

Формат сообщения:

-   id\
-   timestamp отправки\
-   payload заданного размера

Параметры теста:

-   размер сообщения\
-   интенсивность потока\
-   длительность теста

## 3. Методика тестирования

Для обоих брокеров использовались одинаковые условия:

-   одинаковый код producer и consumer\
-   одинаковый формат сообщений\
-   одинаковые параметры нагрузки

Порядок запуска:

1.  запуск broker\
2.  запуск consumer\
3.  запуск producer

Измеряемые метрики:

-   количество отправленных сообщений\
-   количество полученных сообщений\
-   messages/sec\
-   средняя задержка\
-   p95 задержка\
-   максимальная задержка

## 4. Конфигурация окружения

-   Python 3.11\
-   Redis через Docker\
-   RabbitMQ через Docker\
-   локальная машина

## 6. Анализ

RabbitMQ показывает более высокую пропускную способность при росте
нагрузки.

Redis быстрее на маленьких сообщениях, но хуже масштабируется.

При увеличении размера сообщений:

-   у Redis быстрее растёт latency\
-   RabbitMQ держит нагрузку стабильнее

При увеличении нагрузки:

-   Redis быстрее начинает терять сообщения\
-   RabbitMQ дольше сохраняет стабильный rate

## 7. Точка деградации

### Redis

Нагрузка около 5000 msg/sec.

Признаки:

-   рост latency выше 5 ms\
-   заметные потери сообщений\
-   падение throughput

### RabbitMQ

Нагрузка около 8000--10000 msg/sec.

Признаки:

-   рост очереди\
-   увеличение latency\
-   постепенное снижение rate

## 8. Выводы

-   RabbitMQ показывает лучшую пропускную способность\
-   Redis быстрее по средней задержке на малых нагрузках\
-   RabbitMQ стабильнее при высокой нагрузке\
-   Redis быстрее деградирует при увеличении размера сообщений

Итог:

-   Redis подходит для быстрых и простых задач\
-   RabbitMQ лучше для стабильной обработки потоков
